# 멍스쿨 상호작용 감사

## 감사 범위

- 공개 MVP의 버튼 체험 흐름
- 시작 화면 → 명령 입력 → 캐릭터 반응 → 성공/오류 피드백

## 사용자 목표

명령을 내린 순간부터 캐릭터가 반응하고 보상을 얻기까지의 인과관계를 즉시 이해하고, 다시 상호작용하고 싶어야 한다.

## 단계별 상태

1. 시작 상태 — 보통
   - 강아지 원화와 시나리오 안내는 명확하다.
   - 다음 행동은 보이지만 캐릭터의 이름·성장·친밀도처럼 지속 참여를 유도하는 정보가 없다.
   - 증거: `interaction-before-01.png`

2. 명령 입력 — 개선 필요
   - 중앙 피드백과 자세 변화는 작동한다.
   - 버튼, 소리, 진동, 캐릭터 표정, 성장 보상이 하나의 연속된 반응으로 묶이지 않아 입력의 타격감이 약하다.
   - 증거: `interaction-before-02-action.png`

3. 보상 오류 — 개선 필요
   - 무엇이 잘못됐는지는 정확히 설명한다.
   - 긴 오류 문장이 화면 중심을 차지하며, 재시도할 버튼·회복 연출·친밀도 상태가 없어 실패가 단절처럼 느껴진다.
   - 증거: `interaction-before-03-result.png`

## 우선 개선안

- 입력 즉시 명령 예고, 버튼 눌림, 짧은 사운드와 진동을 동시에 제공한다.
- 성공 시 캐릭터 모션, 화면 플래시, 파티클, 친밀도 경험치와 연속 성공을 한 묶음으로 표시한다.
- 실패 시 이유를 짧게 요약하고 기대 명령을 강조해 바로 회복할 수 있게 한다.
- 캐릭터 이름·레벨·친밀도와 보상 타이밍 게이지를 지속 노출한다.
- 시각 효과를 줄이는 환경에서도 텍스트 상태와 접근 가능한 라이브 영역으로 결과를 전달한다.

## 개선 적용 및 재검증

1. 시작 상태 — 개선 완료
   - `멍이`의 레벨, 친밀도, 연속 성공, 최고 기록을 상시 노출했다.
   - 현재 단계에서 필요한 명령만 금색 테두리와 펄스로 강조한다.
   - 증거: `interaction-after-01.png`

2. 행동 입력과 보상 타이밍 — 개선 완료
   - 입력 버튼의 눌림, 명령 텍스트, 강아지 연속 모션, 성공 플래시, 파티클, 효과음·진동을 같은 사건으로 연결했다.
   - 행동 직후 칭찬해야 하는 단계에는 남은 초와 색상 변화가 보이는 타이밍 게이지를 표시한다.
   - 증거: `interaction-after-02-action.png`

3. 상황 완료와 결과 — 개선 완료
   - 상황 완료 시 친밀도 25점을 지급하고, 클리어 콜아웃·성장 수치·확장 링으로 성공을 확실하게 알린다.
   - 결과카드에서 레벨, 친밀도, 연속 성공, 완료율, 정확도, 평균 단계시간을 한 화면에 확인한다.
   - 증거: `interaction-after-03-result.png`, `interaction-after-04-result-card.png`

4. 오류 후 회복 — 개선 완료
   - 긴 오류 설명 대신 `지금은 앉아!`처럼 다음 행동을 짧게 제시하고, 기대 버튼 강조와 연속 성공 초기화를 동시에 확인했다.
   - 증거: `interaction-after-05-recovery.png`

5. 시각 효과 정렬 — 개선 완료
   - 파티클과 보상 링의 중심을 숨겨진 3D 좌표가 아니라 화면에 보이는 강아지 스프라이트 기준으로 계산해 연출이 캐릭터에서 발생하도록 맞췄다.

## 30fps 모션 재제작 감사

### 기존 상태

- 기존 엔진은 `앉아`를 초당 약 4.8장의 키포즈 교체로 처리해, 명령 후 약 270ms 안에 서기에서 앉기로 크게 점프했다.
- 캡처: `motion-before-00.png`~`motion-before-03.png`

### 개선 상태

- 모든 캐릭터 동작을 30fps 시간축으로 다시 구성했다.
- `앉아`, `엎드려`, `손`, `기다려`, `이리와`, `칭찬`에 예비동작·본동작·정착동작을 넣고 위치, 회전, 눌림·늘어남, 그림자 호흡을 프레임마다 보간한다.
- `앉아` 검증에서 프레임 9 → 16 → 23 → 29가 순서대로 재생된 뒤, 보상 대기용 시선 고정 모션으로 자연스럽게 넘어갔다.
- 틀린 명령도 해당 자세를 실제로 수행한 뒤 정답 자세로 돌아가므로 모든 버튼이 캐릭터 반응을 만든다.
- 칭찬 입력은 30프레임 점프·착지 시퀀스와 상황 완료 시퀀스로 연결된다.
- 서로 다른 자세를 투명하게 겹치지 않고 한 키포즈만 표시하도록 해 전환 중 이중 잔상을 제거했다.
- 캡처: `motion-after-10.png`~`motion-after-14.png`, `motion-praise-00.png`~`motion-praise-04.png`, `motion-after-wrong-command.png`
- 잔상 제거 재검증: `motion-after-no-ghost.png`
- 재생 중 브라우저 오류: 0건
